# 테스트 체크리스트

이 문서는 Study Notes Summarizer 애플리케이션의 수동 테스트 가이드입니다.

## 사전 준비사항

1. Supabase 프로젝트 생성 및 설정
   - [ ] Supabase 프로젝트 생성
   - [ ] 환경 변수 설정 (`.env.local` 파일)
   - [ ] SQL 마이그레이션 실행 (`supabase/migrations/` 폴더의 파일들)

2. OpenAI API 키 설정
   - [ ] OpenAI API 키 발급
   - [ ] `.env.local`에 `OPENAI_API_KEY` 설정

3. 개발 환경 실행
   - [ ] `npm install` 실행
   - [ ] `npm run dev` 실행
   - [ ] 브라우저에서 `http://localhost:3000` 접속 확인

## 인증 테스트

### 회원가입
- [ ] `/auth/login` 페이지 접속
- [ ] "계정이 없으신가요? 회원가입" 클릭
- [ ] 유효한 이메일과 비밀번호 입력 후 회원가입
- [ ] 이메일 인증 링크 확인 (Supabase 대시보드에서 확인 가능)
- [ ] 인증 후 로그인 성공 확인

### 로그인
- [ ] `/auth/login` 페이지 접속
- [ ] 등록된 이메일과 비밀번호로 로그인
- [ ] `/dashboard`로 리다이렉트 확인

### 로그아웃
- [ ] 대시보드에서 "로그아웃" 버튼 클릭
- [ ] `/auth/login`으로 리다이렉트 확인

## 파일 업로드 테스트

### 정상 업로드
- [ ] 대시보드에서 `.txt` 파일 선택
- [ ] 파일 업로드 성공 메시지 확인
- [ ] 노트 목록에 새 항목 추가 확인
- [ ] 상태가 "처리 중"으로 표시되는지 확인
- [ ] 몇 초 후 상태가 "완료"로 변경되는지 확인

### 드래그 앤 드롭
- [ ] `.txt` 파일을 드래그하여 업로드 영역에 드롭
- [ ] 업로드 성공 확인

### 파일 유효성 검사
- [ ] `.pdf` 파일 업로드 시도 → 오류 메시지 확인
- [ ] 10MB 초과 파일 업로드 시도 → 오류 메시지 확인
- [ ] 빈 파일 업로드 → 처리 확인

## 요약 기능 테스트

### 자동 요약 생성
- [ ] 텍스트 파일 업로드 후 자동 요약 생성 확인
- [ ] 노트 상세 페이지에서 요약 내용 확인
- [ ] 요약이 한국어로 생성되는지 확인
- [ ] 요약이 원본 내용의 핵심을 담고 있는지 확인

### 요약 재생성
- [ ] 완료된 노트의 "요약 재생성" 버튼 클릭
- [ ] 새로운 요약 생성 확인
- [ ] 실패한 노트의 "요약 재생성" 버튼 클릭
- [ ] 재요약 성공 확인

## 대시보드 테스트

### 노트 목록
- [ ] 업로드한 모든 노트가 목록에 표시되는지 확인
- [ ] 최신 노트가 맨 위에 표시되는지 확인
- [ ] 각 노트의 제목, 날짜, 상태가 올바르게 표시되는지 확인

### 검색 기능
- [ ] 검색창에 노트 제목 입력 → 해당 노트만 필터링 확인
- [ ] 검색창에 요약 내용의 키워드 입력 → 해당 노트 필터링 확인
- [ ] 검색어 삭제 → 모든 노트 다시 표시 확인

### 필터 기능
- [ ] 상태 필터 "완료" 선택 → 완료된 노트만 표시 확인
- [ ] 상태 필터 "처리 중" 선택 → 처리 중인 노트만 표시 확인
- [ ] 상태 필터 "전체" 선택 → 모든 노트 표시 확인

## 노트 상세 페이지 테스트

### 기본 정보 표시
- [ ] 노트 제목 클릭 → 상세 페이지 이동 확인
- [ ] 노트 제목, 생성일, 수정일 표시 확인
- [ ] 상태 배지 표시 확인

### 원본 내용 표시
- [ ] "원본 내용" 섹션에 파일 내용이 올바르게 표시되는지 확인
- [ ] 긴 내용의 경우 스크롤 가능한지 확인

### 요약 내용 표시
- [ ] "요약" 섹션에 요약 내용이 표시되는지 확인
- [ ] 요약이 여러 줄로 표시되는지 확인

### 네비게이션
- [ ] "대시보드로 돌아가기" 링크 클릭 → 대시보드로 이동 확인
- [ ] 상단 로고 클릭 → 대시보드로 이동 확인

## 보안 테스트

### 인증 보호
- [ ] 로그인하지 않은 상태에서 `/dashboard` 접속 시도 → 로그인 페이지로 리다이렉트 확인
- [ ] 로그인하지 않은 상태에서 `/dashboard/notes/[id]` 접속 시도 → 로그인 페이지로 리다이렉트 확인

### 데이터 격리
- [ ] 사용자 A로 로그인하여 노트 업로드
- [ ] 사용자 B로 로그인
- [ ] 사용자 A의 노트가 목록에 표시되지 않는지 확인
- [ ] 사용자 A의 노트 ID로 직접 접근 시도 → 접근 불가 확인
